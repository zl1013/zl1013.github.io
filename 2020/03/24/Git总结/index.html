Git总结

Git学习总结

  • 本地操作流程

  • 工作区（WorkingDirectory）

    • 添加、编辑、修改文件等操作

  • 暂存区

    • 暂存已经修改的文件，最后统一提交到Git仓库

  • Git Repository（Git 仓库）

    • 最终确定的文件保存仓库，成为一个人新的版本，并且对他人可见

常用指令 ：git add 、git commit 、git status

工作区 ---git add --->  暂存区 ---git commit--->Git仓库

  • 本地仓库操作

  • 安装好之后进行全局配置

    $git config --global user.name "username"
$git config --global user.email "email"

  • 选择文件夹进行Git初始化(告诉Git管理此目录)

    git init
    .git隐藏文件夹不能随意更改内容
  • Git命令

  • git status
    查看当前可进行的操作

  • git add

    1.git add  文件名
2.git add  文件名1  文件名2  文件名3  ...
3.git add  .		【添加当前目录到缓存区】

  • git commit

    git commit -m "注释"
  • Git指令

  • 查看版本
    1.git log
    显示Git操作日志

    2.git log --pretty=oneline
    时间+操作 显示

  • 回退操作
    git reset –hard 版本号
    【回到过去之后，再回到未来，则需要使用指令去查看历史操作，以得到未来的commit id】

  • git reflog

  • 远程仓库

  • 两种常规使用方式
  • Use HTTPS
    使用clone指令克隆线上仓库到本地
    `git clone 线上仓库地址`
    在仓库上做对应的操作（提交暂存区、提交本地仓库、提交线上仓库、拉取线上仓库）
    提交到线上仓库指令：git push
    首次线上仓库提交需要授权
版本过低则需要修改.git目录中的config文        件，在remote中的url栗的github.com           前面加上`username:password@`
    拉取线上仓库到本地指令：git pull
  • 每天上班前第一件事 git pull 拉取最新到本地
  • 每天下班最后一件事git push提交最新到线上
  • Use SSH（鉴权方式不同）（比较容易）
    • 创建公私钥对（openssh）
    • ssh-keygen -t rsa -c "email"
    • 三回车
    • 上传公钥到Github
    • 后续操作与HTTPS 相同

  • 分支相关

  • 查看分支：git branch

  • 创建分支：git branch 分支名

  • 切换分支：git checkout 分支名

    • 对于新分支，可以使用git checkout -b 分支名指令来切换分支，-b表示创建并切换，相当于两个指令

  • 删除分支：git branch -d 分支名

  • 合并分支：git merge 被合并的分支名

  • 冲突的产生与解决

  • 先pull到本地

  • 然后解决冲突

  • 重新push

  • 忽略文件

    忽略文件需要新建一个名为.gitignore的文件，用于声明忽略文件或者不忽略文件规则，规则对当前目录及其子目录生效
    该文件没有文件名，只能听过命令行Git bash来 touch创建

1./文件夹名字/过滤整个文件夹
2.*.zip过滤所有.zip文件
3./文件夹名字/do.c过滤某个具体文件
4.!index.php不过滤某个具体文件
